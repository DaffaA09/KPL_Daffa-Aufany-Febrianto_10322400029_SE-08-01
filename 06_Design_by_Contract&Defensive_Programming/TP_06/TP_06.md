## Tugas

Diberikan dua kode yang sama-sama melakukan operasi pembagian. Pertama menggunakan asersi, kedua menggunakan eksepsi.
```js
const assert = require('assert');

function divide(a, b) {
  assert(typeof a === 'number' && typeof b === 'number', 'Nilai harus bilangan bulat');

  assert(b !== 0, 'Tidak bisa pembagian dengan nol');

  return a / b;
}
function divide(a, b) {
  if (typeof a !== "number" || typeof b !== "number") {
    throw new TypeError("Nilai harus bilangan bulat");
  }

  if (b === 0) {
    throw new Error("Tidak bisa pembagian dengan nol");
  }

  return a / b;
}

try {
  const result = divide(10, 2);
  console.log("Hasilnya adalah:", result);
} catch (error) {
  console.error("Error:", error);
}
```
Menurutmu, kapankah kita saatnya menggunakan asersi atau eksepsi untuk fungsi seperti ini di atas? Apakah kita harus sepenuhnya asersi, atau sepenuhnya eksepsi? Lakukan riset dan berikan jawabannya dalam bentuk esai minimal 300 kata.

## Jawaban :

Menurut saya Dalam membuat sebuah program, kita pasti bakal ketemu yang namanya error, misalnya saat membagi angka dengan nol atau ketika input yang diberikan bukan angka. Untuk menangani hal seperti ini, ada dua cara yang sering digunakan, yaitu asersi (assertion) dan eksepsi (exception). Walaupun terlihat mirip karena sama-sama mengecek kesalahan, sebenarnya fungsi keduanya berbeda.

Asersi biasanya digunakan untuk memastikan kondisi yang seharusnya tidak pernah salah. Artinya, asersi dipakai untuk mengecek logika internal dari program itu sendiri. Kalau kondisi asersi gagal, itu menandakan ada bug dalam kode yang kita buat. Karena itu, asersi lebih sering digunakan saat proses development atau debugging oleh programmer. Misalnya, kita yakin suatu variabel pasti berisi angka, maka kita bisa menambahkan asersi untuk memastikan hal tersebut. Jika ternyata tidak sesuai, program akan langsung berhenti agar kita bisa segera memperbaikinya.

Sedangkan eksepsi digunakan untuk menangani error yang memang mungkin terjadi saat program dijalankan, terutama karena input dari pengguna. Contohnya seperti pembagian dengan nol atau input yang bukan angka. Hal-hal seperti ini sangat wajar terjadi, jadi program harus bisa menghadapinya tanpa langsung crash. Dengan menggunakan eksepsi, kita bisa menampilkan pesan error yang jelas kepada pengguna, dan program masih bisa dikontrol menggunakan try-catch.

Jika kita melihat contoh fungsi pembagian tadi, sebenarnya lebih tepat menggunakan eksepsi dibandingkan asersi. Hal ini karena kesalahan seperti input bukan angka atau pembagian dengan nol berasal dari pengguna, bukan dari kesalahan logika program. Jika kita hanya menggunakan asersi, ada kemungkinan program langsung berhenti dan tidak memberikan pengalaman yang baik bagi pengguna.

Namun, bukan berarti asersi tidak berguna. Asersi tetap penting untuk membantu developer menemukan bug lebih cepat saat proses pengembangan. Oleh karena itu, kita tidak perlu memilih salah satu saja. Sebaiknya kita menggunakan kombinasi keduanya sesuai kebutuhan.

Kesimpulannya, asersi digunakan untuk memastikan kondisi internal program tetap benar, sedangkan eksepsi digunakan untuk menangani error yang mungkin terjadi saat program digunakan. Dengan menggabungkan keduanya, program akan menjadi lebih aman, stabil, dan mudah untuk dikembangkan.
