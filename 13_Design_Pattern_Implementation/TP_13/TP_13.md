## Deskripsi 

Pada Tugas Pendahuluan 13, saya menggunakan file repository.ts sebagai contoh penerapan Repository Pattern. Repository Pattern adalah pola desain yang digunakan untuk memisahkan logika akses database dari bagian lain aplikasi, seperti controller atau service. Dengan pola ini, query database dikumpulkan dalam satu tempat sehingga kode menjadi lebih rapi, mudah digunakan ulang, dan lebih mudah dirawat.

Pada file repository.ts, terdapat helper function bernama query<T>() yang digunakan sebagai fungsi umum untuk menjalankan query SQL ke database PostgreSQL. Fungsi ini memakai generics agar hasil query dapat disesuaikan dengan tipe data yang dibutuhkan. File ini juga berisi beberapa repository function seperti findUserByEmail, createUser, findAllKos, findKosById, createPemesanan, createPembayaran, createUlasan, dan fungsi lainnya. Semua fungsi tersebut bertugas mengambil, menambahkan, atau mengubah data dari database.

### Penjelasan Design Pattern

Design pattern yang digunakan adalah Repository Pattern. Pada pola ini, aplikasi tidak langsung menulis query database di banyak tempat. Sebaliknya, semua operasi database dikumpulkan di dalam file repository.

Contohnya:

```ts
async function query<T = any>(sql: string, params: any[] = []): Promise<T[]> {
  if (!pool) {
    throw new Error("Database belum dikonfigurasi (DATABASE_URL kosong)");
  }

  const result = await pool.query(sql, params);
  return result.rows;
}
```
Fungsi tersebut menjadi pusat untuk menjalankan query database. Lalu fungsi lain seperti:
```ts
export async function findUserByEmail(email: string) {
  const rows = await query("SELECT * FROM users WHERE email = $1 LIMIT 1", [
    email,
  ]);
  return rows[0] ?? null;
}
```
menggunakan helper query() agar tidak perlu menulis ulang proses koneksi dan eksekusi query berulang kali.
